# 贡献指南

感谢你愿意参与 ops-triage-assistant！

## 如何贡献

1. **提出问题**：在 Issue 中描述场景与复现步骤。
2. **新增规则**：优先补充规则说明、证据样例、修复建议。
3. **完善文档**：补充 README、Runbook 或架构说明。

## 规则贡献建议

- 保持规则描述清晰、证据可验证。
- 提供可执行的验证命令与修复步骤。
- 如涉及敏感信息，务必脱敏。

## 提交规范

- 建议使用语义化提交信息，例如：
  - `feat: add nginx upstream timeout rule`
  - `docs: update quick start`
  - `chore: move scripts to scripts/`

谢谢你的贡献！
